# React Study Sidebar Restructuring Plan

## 目的
`docs/react-study` 以下の運用を簡素化する。
現在は物理フォルダ（`01-preparation`, `02-jsx-basics` 等）によってサイドバーの構造を維持しているが、ファイルを各フォルダに振り分けるのが手間のため、フラットなファイル配置でも多層的なメニューを実現する。

## 方針
1. **ファイル配置の変更**
   - 現在のサブディレクトリ構成を廃止し、すべての `react_study_XXX.md` ファイルを `docs/react-study/` 直下にフラットに配置する。

2. **サイドバー設定 (`sidebars.ts`) の改修**
   - Docusaurus の `autogenerated` 機能（ディレクトリ構造依存）の使用を、`react-study` については停止または限定的にする。
   - 代わりに、`sidebars.ts` 内でプログラム的にカテゴリ分類を行うロジックを実装する。
   - **ロジック案**:
     - ファイル名の連番（例: `001`〜`010`）を範囲指定し、特定の「Module」や「章」カテゴリに割り振る設定定義を持つ。
     - `sidebars.ts` スクリプトがその定義を読み、自動的にサイドバーのアイテム配列（`items`）を生成する。

## メリット
- 新しい記事を追加する際、適切なフォルダを探して移動させる必要がなくなり、単に連番ファイルを作成して置くだけで済む。
- サイドバーの構成変更（章の区切りの変更など）が、ファイル移動なしに `sidebars.ts` の設定変更だけで完結する。

## 次のステップ
1. 現行ファイルのバックアップ確認。
2. `sidebars.ts` にロジックを実装し、テスト的に少数のフラットファイルで動作確認。
3. 全ファイルをフラット階層へ移動（`git mv` 等を使用）。
4. 動作検証。
